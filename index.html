Javascipt - For adding interactivity to your pages

JavaScript is the programming language that adds interactivity and dynamic behavior to an HTML document, transforming a static page into a responsive user experience.

What is JavaScript used for ?

While HTML defines the structure and CSS handles the styling, JavaScript provides the "brain" by allowing the page to:

  • Respond to user actions like clicks, scrolls, and typing.
  • Update content in real-time without reloading the entire page.
  • Manipulate elements directly through the Document Object Model (DOM).

Linking

A modern way of linking an external JavaScript file is to place the <script> element in the <head> section and use the defer or async attribute to prevent blocking HTML parsing.

Use defer for scripts that depend on the DOM or when execution order matters.
Use async otherwise (for independent scripts).

But in modern application code, the preferred approach is to use type="module".

Module scripts are deferred by default, execute after HTML parsing, preserve execution order, and support import/export, making them cleaner and better structured.

async remains useful for independent third-party scripts (analytics, ads), and defer is still relevant for classic (non-module) scripts that rely on the DOM.

Example using 'type=module', to be added at the end of the <head> section:

<script type="module" src="app.js" ></script>

Note: ES modules may not always load, for example when opening a local HTML file directly in the browser, because browsers block module scripts for security reasons.

Publish a site for free via Github Pages

The steps to publish a site via Github are as follow:

  1. Create an account
    • Go to Github
    • Create a free account
  2. Create a repository
    • Click on New Repository
    • Choose a name. For example: my-test-site.
    • Public
    • Add a README
    • Create repository
  3. Add the site's files
    • Click on Add file
    • → Upload files
    • Add all the files
      Note: One way is to have the files dropped all at once from the explorer
    • Commit changes
    ⚠️ Important: The main access file must be named 'index.html'.
  4. Activate GitHub Pages
    • Click on Settings
    • Menu de gauche, choisir Pages
    • Source = "Deploy from a branch"
    • Branch = main
    • Save.
    ⏳ Wait for one or two minutes

    A new URL appears in the page

  5. Test on remote device
    • Open the URL on the remote device
    • There we are. When uploading new files, after committing, the site updates automatically.

Styling Links: A Guide to CSS Pseudo-Classes

In CSS, a pseudo-class is a keyword added to a selector that specifies a special state of the element being selected.

Links (the <a> tag) go through several states as users interact with them. These states are targeted using pseudo-classes to provide visual feedback and improve accessibility for keyboard and screen reader users.

The five primary link states are:

  • :link (Unvisited): The default state for a link that has not been clicked yet. By default, browsers typically display these as blue and underlined.
  • :visited: A link the user has previously opened. To protect user privacy, browsers limit the CSS properties you can change here (mostly just colors).
  • :hover: Triggered when a user moves their mouse pointer over the link without clicking it. This provides a visual cue that the element is interactive.
  • :focus: Occurs when a link is selected via keyboard navigation (usually the Tab key). This is critical for accessibility to show keyboard users where they are on the page.
  • :active: The momentary state while a link is actually being clicked or activated. In default browser styling, these often turn red.

The "LVFHA" Ordering Rule

Because later CSS rules can override earlier ones, you must define these in a specific order to ensure they work correctly. A common mnemonic for this is "LoVe Fears HAte":

  1. Link
  2. Visited
  3. Focus
  4. Hover
  5. Active

If you define :visited after :hover, for example, the hover effect might never appear for links the user has already seen.
